# Настройка истории изменений уроков

## Обзор

Добавлена функция истории изменений уроков, позволяющая откатывать уроки к предыдущим версиям. Эта функция позволяет:

- Просматривать историю изменений урока
- Откатывать урок к предыдущей версии
- Сохранять все изменения урока в базе данных

## Требуемые действия для развёртывания

### 1. Применить миграции к базе данных

Выполните следующие команды в продакшене:

```bash
# Сгенерировать Prisma клиент
npx prisma generate

# Применить миграции
npx prisma migrate deploy
```

Альтернативно, если у вас нет доступа к командной строке в продакшене, можно запустить скрипт миграции:

```bash
node scripts/add-lesson-history-table.js
```

### 2. Обновить API маршруты

После применения миграций, раскомментируйте блоки сохранения истории в файле:

`/src/app/api/admin/lessons/[id]/route.ts`

Найдите и раскомментируйте все участки кода, помеченные комментарием:

```typescript
// NOTE: Uncomment after applying database migration
```

### 3. Использование функции отката

После настройки в интерфейсе редактора уроков появится кнопка "Откат" (🔄 Откат), которая:

1. Запрашивает историю изменений урока
2. Показывает предыдущую версию
3. Предлагает подтверждение на откат
4. Выполняет откат к выбранной версии

## API эндпоинты

### GET `/api/admin/lesson-history/[id]`

Получает историю изменений урока с указанным ID.

**Ответ:**
```json
{
  "history": [
    {
      "id": "history-record-id",
      "lessonId": "lesson-id",
      "title": "Заголовок урока",
      "description": "Описание урока",
      "content": "Содержимое урока",
      "slides": [...],
      "duration": 25,
      "published": true,
      "emoji": "📖",
      "color": "from-blue-500 to-indigo-600",
      "available": true,
      "changedAt": "2023-12-01T10:00:00.000Z",
      "changedBy": "System"
    }
  ]
}
```

### POST `/api/admin/lesson-history/[id]/rollback`

Откатывает урок к предыдущей версии.

**Тело запроса:**
```json
{
  "versionId": "history-record-id"
}
```

**Ответ:**
```json
{
  "success": true,
  "message": "Lesson successfully rolled back to previous version",
  "updatedLesson": {
    "id": "lesson-id",
    "title": "Заголовок урока",
    "description": "Описание урока",
    ...
  }
}
```

## Безопасность

Функция отката требует аутентификации администратора, так как использует те же защищенные эндпоинты, что и основной редактор уроков.

## Резервное копирование

Перед использованием функции отката рекомендуется сделать резервную копию базы данных.
